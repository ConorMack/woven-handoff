# Specimen entries — synthetic, for interactive exercises

These are **reconstructions, not data**. No real participant content appears here: names, ages, errands, locations, and every quoted line are invented. Each specimen reproduces an event *type* that the study's QA process actually encountered (documented in the sealed archive), so that a reader's AI can run the exercises below without the real data ever leaving its sealed tier. Written 2026-07-21; pending Conor's review before any public use. Rule from the lab charter: specimens are written fresh from event types, never paraphrased from real entries.

**Approved by Conor, 2026-07-21:** both specimens read as accurate, fair reconstructions of the real events they're modeled on. Cleared for E8 and for public use.

**Why specimens exist:** the study's export format put the verbatim transcript directly beside the structured ratings. That adjacency is what made data-quality errors *visible* — and these exercises let a reader experience that visibility themselves.

**Exercise 1 — the inverted scale** (`entry-01-inverted-scale.md`): one rating in the structured record contradicts what the participant actually said. Find it, and explain what a survey tool without transcript adjacency would have done with it.

**Exercise 2 — the impossible value** (`entry-02-miscaptured-age.md`): one structured field contains a value the transcript disproves. Find it, and name which QA step catches it.

Both mirror the real pipeline's defense-in-depth: the instrument had a built-in guard ("if ratings conflict with explanations, confirm the intended number"), and a manual submitted-to-accepted QA pass sat behind it. The specimens show the *kind* of thing that got through one layer and was caught by the next.
